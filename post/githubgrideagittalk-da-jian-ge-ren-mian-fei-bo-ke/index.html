GitHub+Gridea+GitTalk 搭建个人免费博客

2024-09-23 3 min read # Gridea

官方文档链接:https://open.gridea.dev/docs
这里涵盖了Gridea遇到的多数问题：https://open.gridea.dev/faq/
这里涵盖了GitTalk 遇到的多数问题：https://github.com/gitalk/gitalk/issues
必要条件：下载安装Gridea，有GitHub账号，相信你能看到这里证明是具备账号的。

  1. 仓库创建
    首先，需要创建一个仓库，建议仓库为 http://username.github.io。根据官方文档， GitHub 会默认将名称符合要求的仓库设置成 GitHub Pages 可访问类型。例如我的用户名是 duanxinyua，所以仓库名就是duanxinyua.github.io。如果遇到创建仓库的时候并未自动启用 GitHub Pages，遇到这种情况手动开启即可。仓库名设置为[用户名.github.io] ，如下图所示，已经设置过了，所以显示重复，输入仓库名称点击右下角创建仓库。

  2. 开启GitHub Pages和自定义域名
    创建仓库的时候名称符合会自动启用，没有的话点击仓库的设置（Settings），如图点击页面（Pages），列表第一项开启即可。如果有自定义域名填写到设置中，如图的Custom domain，记得在你所在的域名服务商做域名解析，我这里是www通过cname解析到仓库名：duanxinyua.github.io。

  3. 在设置(Setting)界面将部署的页面设置为main，之后要填写到Gridea配置中，记得保存！！！如下图所示。（到此仓库创建完成）

  4. 创建用于Gridea连接的Token
    在个人Setting界面创建用于Gridea连接的Token，点击头像，选择设置（Settings），点击开发人员设置（Developer settings），个人访问令牌（Personal access tokens），令牌（Tokens (classic)），生成新令牌（Generate new token（classic）），repo权限即可，别的权限不要防止泄露，记得保存好Token，关闭页面后就看不见了。

  5. 配置Gridea
    下载Gridea：https://open.gridea.dev/#started，根据自己的系统选择安装，
    软件界面远程设置一下，按照下图设置自己的仓库就行。

  6. 配置GitTalk（评论）
    跟上面Token类似，OAuth应用程序（OAuth Apps），生成一个用来授予GitTalk的ID和secret，基础仓库记得加上 https://，结尾也要加上/。如果你使用的是自定义域名，这里填写你的自定义域名。

    然后将ID和secret上传到Gridea软件。
